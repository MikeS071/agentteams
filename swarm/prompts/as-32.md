# AS-32: Swarm panel — live execution progress detail

## Context
AS-16 built the collapsible swarm status panel (compact + full mode). This ticket enhances the **Running** section with real-time progress detail so users can see what's actually happening during execution.

## Scope
Enhance `SwarmStatus.tsx` (and any backing API/polling) to show granular execution state.

## Requirements

### Running Section (expanded panel)
For each currently running ticket, display:
- **Ticket ID + title** (e.g. `as-25: Quick Action framework`)
- **Status badge**: animated "Running" indicator (blue pulse dot)
- **Elapsed time**: live counter since agent started (e.g. `3m 42s`), updates every second
- **% complete**: progress bar if available from agent output parsing (optional — show indeterminate spinner if unavailable)
- **Current step**: last meaningful log line or phase step from agent (e.g. "Running tests...", "Writing component...")
- Clicking a running ticket expands to show last ~10 lines of agent output (tail of tmux capture)

### Queued / Pending Section
Below running items, show a "Next Up" section:
- List all `todo` status tickets in current phase, ordered by dependency
- Each shows: ticket ID, title, dependency status (e.g. "waiting on as-25")
- Tickets with all deps met show "Ready" badge
- Tickets in future phases shown as collapsed group: "Phase 3: 5 tickets pending"

### Data Flow
- Poll `/api/swarm/status` endpoint every 5 seconds (or use SSE if already available)
- API reads from `swarm/tracker.json` + optionally captures tmux pane output for running agents
- Include `started_at` timestamp in tracker entries for elapsed time calculation
- Parse agent output for progress hints (test counts, build steps) to estimate % complete

### Compact Mode Update
Update the collapsed single-row to include the name of the current running ticket:
`▼ Swarm: agentsquads  ████░░ 3/12 done · as-25 running (2m 15s)  [✕]`
If multiple running: `· 2 running (as-25, as-26)`

## Files to modify
- `apps/web/src/components/SwarmStatus.tsx` — add RunningDetail + QueuedList sub-components
- `apps/web/src/app/api/swarm/status/route.ts` — extend response with `started_at`, agent output tail
- Possibly `apps/web/src/hooks/useSwarmStatus.ts` — polling hook with 5s interval

## Acceptance Criteria
- [ ] Running tickets show elapsed time (live counter, updates every second)
- [ ] Running tickets show % complete bar (or indeterminate if unavailable)
- [ ] Running tickets show last meaningful agent output line
- [ ] Click-to-expand shows last 10 lines of agent output
- [ ] Queued section shows todo tickets with dependency status
- [ ] Future phase tickets collapsed into summary groups
- [ ] Compact mode shows current running ticket name + time
- [ ] No performance regression — polling doesn't block UI
