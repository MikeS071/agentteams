# AS-33: Auto-reload status views on tracker changes

## Context
When new tickets are added (e.g. as-32 was added to `swarm/prompts/` but didn't appear in the TUI or dashboard swarm view). Status views should pick up tracker/ticket changes automatically without manual refresh.

## Scope
Both the CLI TUI (`agent-swarm status --watch`) and the AgentSquads dashboard swarm panel must auto-detect and reload when the underlying data changes.

## Requirements

### CLI TUI (`agent-swarm status --watch`)
- Watch `swarm/tracker.json` for filesystem changes (fsnotify or poll every 3s)
- On change: re-read tracker, re-render table with new/removed/updated tickets
- Also watch `swarm/prompts/` directory — new `.md` files = new tickets available
- Show brief flash/highlight on rows that changed since last render

### Dashboard Swarm Panel (SwarmStatus.tsx)
- The `/api/swarm/status` endpoint should return a `revision` or `lastModified` timestamp
- Frontend polling (already 5s from as-32) compares revision — if changed, full re-render
- New tickets appear automatically in the Queued/Pending section
- Removed tickets disappear
- Status transitions (todo→running→done) animate smoothly

### Edge Cases
- Ticket added to `prompts/` but not yet in `tracker.json` → show as "unregistered" or trigger auto-register
- `tracker.json` deleted or corrupted → show error state, don't crash
- Rapid successive changes → debounce (200ms for TUI, already handled by poll interval for web)

## Files to modify
- `agent-swarm` CLI: `internal/tui/` or `cmd/status.go` — add file watcher
- `apps/web/src/app/api/swarm/status/route.ts` — add revision/etag to response
- `apps/web/src/components/SwarmStatus.tsx` — compare revision, re-render on change
- `apps/web/src/hooks/useSwarmStatus.ts` — revision-aware polling

## Acceptance Criteria
- [ ] CLI TUI auto-updates within 3s when tracker.json changes
- [ ] CLI TUI detects new prompt files in swarm/prompts/
- [ ] Dashboard swarm panel shows new tickets within one poll cycle (5s)
- [ ] Status transitions render without full page reload
- [ ] Corrupted/missing tracker shows error state, doesn't crash
